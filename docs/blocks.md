# tfreport Block Reference

Auto-generated by `cmd/docgen` from the `Block.Doc()` metadata of each registered block. Do not edit by hand; run `make docs` to regenerate.

See [docs/output-templates.md](output-templates.md) for composition tiers, raw-data escape hatch, and Sprig functions.

## Blocks

### `changed_resources_table`

Per-resource impact table (`| Resource | Name | Changed | Impact |`). Used inside github-step-summary instance dropdowns.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `actions` | `csv` | update,delete,replace | Filter by action. Use `all` to include create and read. |
| `max` | `int` | 0 (no limit) | Cap number of rows. Hits emit a `… N more resources` marker. |


### `deploy_checklist`

GitHub task-list checkboxes, one per report. Degenerates to a single item in single-report mode.


### `diff_groups`

Collapses resources with identical change fingerprints into grouped rows. Single-report only; use fleet_homogeneity for cross-report uniformity.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `threshold` | `int` | 2 | Only collapse when group size ≥ threshold. |
| `actions` | `csv` | update,delete,replace | Which actions participate in fingerprint grouping. |


### `fleet_homogeneity`

Detects uniformity across multi-report renders. When all reports match, summarizes once; when they diverge, flags outliers against the majority pattern.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `style` | `string` | summary | One of `summary`, `banner`, `table`. |
| `fingerprint` | `string` | key_changes | Equality strategy: `key_changes` (text-based) or `action_counts` (coarser, shape-based). |


### `footer`

Closing credit line and data-source-read footnote. Target-aware; markdown renders minimal, GitHub targets add the generator credit.


### `glossary`

Opt-in blockquote defining tfreport's action + impact vocabulary. Never in default templates.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `include` | `csv` | actions,impacts | Which sections to render. Any subset of `actions`, `impacts`, `imports`. |
| `level` | `string` | beginner | `beginner` (verbose explanations) or `intermediate` (compact). |


### `instance_detail`

Per-module-instance detail sections used by github-step-summary: collapsible headers wrapping impact tables plus text-plan or synthetic diffs.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `show` | `csv` | impact_table,diff | Which inner sections to include. Any subset of `impact_table,diff`. |
| `group_submodules` | `bool` | (from ctx.Output.GroupSubmodules) | Override config; when true, sub-modules render as nested <details>. |
| `max` | `int` | (from ctx.Output.MaxResourcesInSummary, else 50) | Cap number of instances shown. 0 means unlimited. |


### `key_changes`

Plain-English summary bullets, impact-tagged, with optional filter and truncation.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `max` | `int` | 0 (no limit) | Cap number of bullets; extras collapse into a `… N more changes` line. |
| `impact` | `csv` | (all) | Filter: keep only entries whose Impact is in the csv set (e.g. `critical,high`). |


### `module_details`

Per-module section: collapsible <details> (GitHub targets) or flat H3 (markdown) wrapping a resource table or per-resource diff block.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `per_resource` | `bool` | true for github-pr-comment, else false | When true, emits a ```diff code block with one line per resource instead of a resource table. |


### `modules_table`

Flat one-row-per-module-group markdown table with pluggable columns. Pick columns, optionally cap rows.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `report` | `*core.Report` | (current report) | Explicit report to render. Required when looping range .Reports; pass $r. |
| `columns` | `csv` | module,changed_attrs | Comma-separated column IDs to include. See Columns below. |
| `max` | `int` | 0 (no limit) | Cap the table at this many rows. Extra rows collapse into a single '…' row. |
| `empty` | `string` | — | Cell value used for empty/missing data. |

**Columns** (for the `columns` csv arg):

| ID | Heading | Description |
|----|---------|-------------|
| `actions` | Actions | Action summary line (e.g. `2 update, 1 create`). |
| `changed_attrs` | Changed attributes | Union of all changed attribute keys in the group. |
| `description` | Description | Team-supplied module description from config or presets. |
| `impact` | Impact | Worst impact across the group, with emoji. |
| `module` | Module | Module call name (ModuleGroup.Name). |
| `module_path` | Module path | Full dotted module path (e.g. `module.vnet.module.subnet`). |
| `module_type` | Module type | Module type derived from the source URL (e.g. `virtual_network`). |
| `resources` | Resources | Count of resource changes in the group. |

**Example:**

```tmpl
{{ modules_table "report" $r "columns" "module_type,module,changed_attrs" }}
```

Renders:

```
| Module type | Module | Changed attributes |
|---|---|---|
| `virtual_network` | `vnet` | `address_space`, `tags` |
```


### `per_report`

One 'report card' section for a single report. Declarative replacement for the {{ range .Reports }}<details>…{{ end }} loop in multi-report templates. Never wraps title/plan_counts/footer (those are top-level chrome).

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `report` | `*core.Report` | — | Required. The report to render; pass `$r` from range .Reports. |
| `show` | `csv` | key_changes | Inner blocks to compose: key_changes, summary_table, module_details, changed_resources_table, text_plan, instance_detail. |
| `collapse` | `bool` | (target uses <details>) | Force wrap/unwrap in <details>. Rarely set explicitly. |


### `plan_counts`

Terraform-style verb summary (`Plan: 1 to add, 2 to change, 1 to destroy.`). Target-agnostic. Empty plan renders `No changes detected.`


### `risk_histogram`

Impact-level distribution across all resources in scope. Three visual styles (bar table, inline one-liner).

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `style` | `string` | bar | One of `bar` (table with unicode bars), `table` (no bar column), `inline` (single line). |
| `include_none` | `bool` | false | Include `impact=none` (no-op) in output. |
| `max_bar` | `int` | 40 | Cap bar length; counts above this truncate with a `+` marker. |


### `summary_table`

Top-level resource-count table with five grouping modes. Each grouping accepts its own subset of columns via the `columns` csv arg (defaults to the full column set for the grouping).

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `group` | `string` | (target-dependent) | One of `module`, `module_type`, `subscription`, `action`, `resource_type`. |
| `columns` | `csv` | (full set for the grouping) | Column subset to render. Valid IDs depend on `group` — see Columns below. |
| `hide_empty` | `bool` | false | Drop rows with zero non-read resources. |
| `max` | `int` | 0 (no limit) | Cap number of rows. No effect for `group=action`. |

**Columns** (for the `columns` csv arg):

| ID | Heading | Description |
|----|---------|-------------|
| `action` | Action | group=action: the terraform action with emoji. |
| `actions` | Actions | group=module,module_type,resource_type,subscription: action-breakdown summary string. |
| `add` | Add | group=subscription (pr-comment default): create count. |
| `count` | Count | group=action,resource_type: resource count for the row. |
| `delete` | Delete | group=subscription (pr-comment default): delete count. |
| `description` | Description | group=module_type: team-supplied module description (auto-hidden when empty across all rows). |
| `impact` | Impact | group=action,subscription: natural/worst impact for the row. |
| `instances` | Instances | group=module_type: count of distinct top-level module instances. |
| `module` | Module | group=module: the module call name. |
| `module_type` | Module Type | group=module_type: resolved module type from source URL. |
| `replace` | Replace | group=subscription (pr-comment default): replace count. |
| `resource_type` | Resource Type | group=resource_type: display name + raw type in backticks. |
| `resources` | Resources | group=module,module_type,subscription: non-read resource count. |
| `subscription` | Subscription | group=subscription: the report label. |
| `update` | Update | group=subscription (pr-comment default): update count. |

**Example:**

```tmpl
{{ summary_table "group" "module" "columns" "module,resources" }}
```

Renders:

```
| Module | Resources |
|---|---|
| vnet | 4 |
| nsg | 3 |
```


### `text_plan`

Native terraform plan text block, budget-aware. Truncates at newline boundaries when ctx.TextBudget would be exceeded.

**Args:**

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `addresses` | `csv` | (all resources in report) | Restrict to these resource addresses; empty renders every address with a text block. |
| `fence` | `string` | (from ctx.Output.CodeFormat) | Override code fence language: `diff`, `hcl`, `terraform`, or any other for plain. |


### `title`

Target-aware plan header. Zero-arg; grammar switches on ctx.Target (markdown H1, pr-body/pr-comment/step-summary each distinct).
